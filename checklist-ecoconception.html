Checklist d'Écoconception

L'outil de synthèse pour vos projets

Utilisez cette checklist comme un guide pratique pour évaluer vos services numériques. Elle regroupe les bonnes pratiques essentielles à appliquer lors des phases de conception, de développement et de test. C'est la base de l'évaluation de la maturité (le "Radar").

Commencer l'évaluation

Checklist des Bonnes Pratiques

Passez en revue chaque point avec votre équipe et cochez ce qui est déjà appliqué sur votre périmètre.

Bonne Pratique Fait ? (OUI/NON) Fiche Pratique Détaillée

Limiter le nombre de webfonts utilisées Voir la fiche
Utiliser les formats de police WOFF2 ou WOFF Voir la fiche
Minimiser le nombre d'animations javascript et CSS Voir la fiche
Désactiver l'autoplay des vidéos Voir la fiche
Désactiver l'autoplay des audios Voir la fiche
Limiter (ou supprimer) les carrousels automatiques Voir la fiche

Catégorie : Back-end & Architecture

Bonne Pratique Fait ? (OUI/NON) Fiche Pratique Détaillée
Utiliser le protocole HTTP/2 ou HTTP/3 Voir la fiche
Mettre en place une stratégie de cache serveur efficace
Optimiser les requêtes d'accès aux données (limiter les jointures, N+1...)
Limiter les données transmises (pagination, champs API ciblés)
Optimiser les flux HTTP (compression Gzip/Brotli)
